Version A A

사용자가 말하게 한다.

  • 2개 질문으로 사용자의 진지함과 결혼관을 받음

Version B B

  • 한 화면에 결하다의 정체성, 약속, 검증을 모두 담음
  • "동의하고 시작하기" 한 번의 클릭으로 진입
  • Self-Selection 필터 효과 (진지하지 않은 사용자 자연 이탈)
  • 약 10초 소요
Version A · 강점과 약점

📝 Question Style

강점:

① 자기설득의 힘. 사용자가 직접 손으로 "결혼을 진지하게 준비 중"을 선택하면, 그 선택과 일관되게 행동하려는 심리가 작동합니다. Cialdini의 일관성 원칙. 가입 후 이탈률이 낮아집니다.

② 데이터 수집. 2개 답변이 매칭 알고리즘의 사전 가중치로 활용 가능합니다. 24문항 결과의 prior로 작동해서 매칭 정밀도를 직접 끌어올립니다.

③ 맞춤화 신호. "이 앱이 내 답을 듣고 시작한다"는 인상이 매니저 포지셔닝의 첫 증명이 됩니다.

약점:

① 마찰이 더 큼. 30초가 걸리고, 두 번의 결정이 필요합니다. 가입 전 0커밋 사용자에게는 부담일 수 있습니다.

② 24문항과 톤이 비슷함. 비록 인증서류 버퍼가 있긴 하지만, 사용자가 "또 질문이네"라고 느낄 가능성이 0은 아닙니다.

🎯 적합한 사용자

이미 결하다를 어느 정도 알고 들어온 사용자, 매칭 앱 경험이 있어 "내 정보를 입력하면 정밀한 매칭이 된다"는 기대가 있는 사용자.

Version B · 강점과 약점

🤝 Acknowledgement Style

강점:

① 강력한 첫인상. "결혼을 전제로 한 만남"이라는 선언이 첫 화면에 등장하면, 사용자는 즉시 결하다의 정체성을 이해합니다. 다른 데이팅 앱과의 차별점이 첫 1초에 각인됩니다.

② 강한 Self-Selection 필터. 결혼 의지가 없는 사용자는 "동의하고 시작하기" 버튼을 누르기 전에 스스로 이탈합니다. 진지한 사용자만 가입 단계로 넘어가므로 전체 풀의 질이 올라갑니다.

③ 약속 카드의 신뢰 구축. "검증 / 매니저 / 가입비 0원" 세 가지 약속이 한 화면에 등장하면서 진입 직후 신뢰가 즉시 형성됩니다.

④ 마찰이 매우 낮음. 10초 안에 끝납니다. 클릭 한 번이면 됩니다.

약점:

① 경고성 인상 리스크. "가벼운 만남은 사양합니다" 같은 강한 표현은 정당하지만, 첫 화면에서 너무 필터링하는 느낌을 줄 수 있습니다. 창완님 우려대로 진성 회원까지 거부감을 느끼면 회원 풀 자체가 줄어듭니다.

② 자기설득 효과 0. 사용자는 동의 버튼만 누를 뿐 자기 의사를 표현하지 않습니다. Cialdini의 일관성 원칙이 거의 작동하지 않습니다.

③ 데이터 수집 0. 매칭 알고리즘에 활용할 prior 데이터가 없습니다. 진지함과 결혼관을 24문항에서 다시 처음부터 측정해야 합니다.

🎯 적합한 사용자

결하다를 처음 들어본 사용자, 다른 데이팅 앱과 결하다를 혼동할 가능성이 있는 사용자, 빠른 진입을 선호하는 사용자.

⚠️ 톤 다운 적용

Version B의 카피는 정훈님 의견과 창완님 우려를 모두 반영해서 부드럽게 다듬었습니다. "사양합니다" 같은 거부 표현 대신 "가벼운 만남이나 데이팅이 아니라, 진지하게 인연을 잇는 일을 함께해요"로 변경. 같은 메시지를 환영의 톤으로 전달합니다. 그리고 하단에 "지금이 맞는 시기가 아니라면 다음에 다시 와주세요"라는 부드러운 출구를 마련해서 거부감을 최소화했습니다.

가설

🔬 검증 가설

H0 (귀무가설): Version A와 Version B의 다운로드 → 가입 완료 전환율은 차이가 없다.

H1 (대립가설): 두 버전 사이에 유의미한 전환율 차이가 존재한다.

핵심 질문: 결하다 사용자에게 더 효과적인 것은 "답하게 만드는 것"인가, "정체성을 알리는 것"인가?

측정 지표

1차 지표 · 가입 완료 전환율

다운로드 후 회원가입(전화번호 인증)까지 도달하는 비율. 가장 직접적인 효과 측정.

전환율 = 가입 완료 사용자 수 / 다운로드 사용자 수

2차 지표 · 24문항 도달률

가입 후 인증서류와 자기소개를 거쳐 24문항 검사를 완료하는 비율. 진성 회원의 비율을 보여줌.

도달률 = 24문항 완료 / 가입 완료

3차 지표 · 첫 매칭 수락률

매니저의 첫 추천을 수락하는 비율. 사용자의 진지함과 매칭 만족도를 동시에 보여줌.

수락률 = 매칭 수락 수 / 매칭 제안 수

4차 지표 · 30일 잔존율

가입 후 30일 시점에 활동하는 사용자 비율. Self-Selection 효과의 장기 검증.

잔존율 = 30일 활성 사용자 / 가입 완료

5차 지표 · 첫 미팅 성사율

가장 중요한 비즈니스 지표. 결하다의 매출과 직결.

성사율 = 실제 미팅 발생 수 / 가입 완료
실행 방법

⚙️ 테스트 운영

1. 트래픽 분배: 신규 다운로드 사용자를 50:50으로 무작위 배정. 사용자별로 한 버전만 노출되며, 같은 사용자가 두 버전을 모두 보는 일은 없음.

2. 테스트 기간: 최소 2주, 권장 4주. 통계적 유의성을 확보하려면 각 버전당 최소 500명 이상의 가입 시도가 필요.

3. 분리 기준: 디바이스 ID 기반 무작위 분배. 같은 사용자가 앱을 재설치해도 같은 버전이 노출되도록 ID 기반 시드 고정.

4. 변경 금지 항목: 테스트 기간 동안 온보딩 5장, 인증서류, 24문항은 동일하게 유지. 두 그룹이 같은 조건에서 비교될 수 있도록.

5. 조기 종료 조건: 한 버전의 가입 완료 전환율이 다른 버전 대비 30% 이상 낮으면 1주차에 조기 종료 검토.

📊 기대 결과 시나리오

시나리오 1 — A 우세: 자기설득 효과가 강하게 작동하는 경우. 가입 완료율은 비슷하지만 24문항 도달률과 첫 매칭 수락률이 A에서 더 높을 가능성.

시나리오 2 — B 우세: Self-Selection 필터가 강하게 작동하는 경우. 가입 완료율 자체는 B가 낮을 수 있지만, 24문항 도달률과 30일 잔존율이 B에서 훨씬 높게 나타날 가능성.

시나리오 3 — 혼합: 1차 지표(가입 완료율)에서는 A가 유리하지만, 5차 지표(첫 미팅 성사율)에서는 B가 유리한 경우. 이 경우 비즈니스 우선순위(빠른 성장 vs 매출)에 따라 결정.

의사결정 프레임

🎯 결과를 어떻게 해석할 것인가

단순히 가입 전환율만 보면 안 됩니다. 결하다는 가입자 수가 아니라 매칭 성공률이 핵심입니다. 그래서 다음 우선순위로 의사결정합니다.

1️⃣
최우선: 5차 지표(첫 미팅 성사율)에서 명확히 우세한 안 채택. 매출 직결.
2️⃣
차순위: 5차 지표가 비슷하면 4차 지표(30일 잔존율) 우세 안 채택. 진성 회원 풀.
3️⃣
그 다음: 4차도 비슷하면 1차 지표(가입 완료 전환율) 우세 안 채택. 트래픽 효율.
🤝
하이브리드 옵션: 두 안의 강점을 결합. B의 정체성 선언을 짧게 보여준 후 A의 2개 질문을 던지는 3단계 구조도 가능. 단, 마찰이 늘어나는 리스크 검증 필요.
